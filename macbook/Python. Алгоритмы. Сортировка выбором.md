 Алгоритм сортировки выбором - хороший инструмент для сортировки с использованием дополнительного списка (заранее пустого), который выполняется за *O(n2) - квадратичное время*

>[!note] Массивы
>Массивы хороши в том случае, если вы не собираетесь их изменять в дальнейшем, ведь в таком случае происходит огромное кол-во операций с исходным массивом, вследствие чего он по факту становится новым, тоесть пересоздается заново. Зато данные штуковины очень хорошо подходят для регулярного из них чтения - т.к элементы известны заранее

>[!note] Связанные списки
>Связанные списки работают по принципу - каждый список хранит в своей памяти место памяти следующего по цепочке связанного списка. Они хорошо подходят для различных операций - удаления, редактирования, вставки, но не подходят для чтения, т.к поиск элемента проходит последовательно - от самого первого к самому последнему
