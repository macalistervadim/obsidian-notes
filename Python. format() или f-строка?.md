
В Python существует несколько способов форматирования строк, основные из них:

- `f-строки` (f-strings)
- метод `.format()`

Оба способа используются для **подстановки значений в строку**, однако отличаются механизмом работы и сценариями применения.

---

### Общая идея

> Оба подхода решают одну задачу — формирование строки с подставленными значениями.

Пример:

name = "Vadim"  
  
# f-string  
f"Hello, {name}"  
  
# format  
"Hello, {}".format(name)

---

###  Ключевое различие

#### `f-строки`

- вычисляются **на этапе выполнения строки (inline)**
- требуют, чтобы переменные были доступны **в текущем контексте**

f"{value}"

👉 выражение внутри `{}` сразу вычисляется

---

#### `str.format()`

- работает через **шаблон + подстановка аргументов**
- значения можно передать **отдельно**

"{}".format(value)

👉 строка и данные могут существовать независимо

---

### Когда что использовать

#### 1. `f-строки` — основной выбор

Используются в большинстве случаев:

- читаемость выше
- компактнее
- поддерживают выражения

f"{user.name} has {len(items)} items"

👉 это стандарт де-факто в современном Python

---

#### 2. `str.format()` — когда нужен шаблон

Полезен, если:

- строка задаётся заранее (например, в конфиге или шаблоне)
- форматирование происходит **позже**
- нет прямого доступа к переменным в момент объявления строки

template = "User: {name}, age: {age}"  
result = template.format(name="Vadim", age=20)

---

### Важное ограничение `f-строк`

`f-строки` нельзя использовать, если:

- строка формируется динамически (например, загружается из файла)
- переменные недоступны в момент создания строки

👉 в этих случаях `.format()` или другие методы — единственный вариант

---

### Частые ошибки

- считать, что `format()` «устарел» → это не так
- использовать `format()` там, где проще f-string
- не понимать разницу между **шаблоном и вычислением**

---

###  Важное замечание (для бэкенда)

> `f-строки` нельзя использовать для формирования SQL-запросов

# ❌ опасно  
f"SELECT * FROM users WHERE id = {user_id}"

👉 это приводит к SQL Injection

---

###  Краткое резюме

>[!tip]
>- `f-строки` — основной инструмент, когда данные доступны сразу
>- `.format()` — полезен для шаблонов и отложенного форматирования
>- разница не только в синтаксисе, но и в **модели использования**
